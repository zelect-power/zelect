# Техническое задание

## Корпоративный сайт-каталог Zelect Power Technology

**Домен:** zelect.com.ua
**Версия ТЗ:** 1.0
**Бюджет проекта:** 500 USD (фиксированная сумма)

---

## 1. О проекте

Корпоративный сайт-каталог для компании **Zelect Power Technology** — поставщика промышленных трансформаторов и электрооборудования. Целевая аудитория — промышленные предприятия и государственные структуры. Сайт должен быть современным, строгим и удобным, без лишних элементов.

### Базовые параметры

| Параметр | Значение |
|---|---|
| Домен | `zelect.com.ua` |
| Тип сайта | Корпоративный сайт-каталог (без корзины в первой версии) |
| Основной язык | Украинский (первая версия — только `uk`) |
| Стек | Next.js 15+ (App Router) + Payload CMS 3+ + PostgreSQL |
| Тема | Светлая (по умолчанию) + переключатель на тёмную |
| Референс дизайна | [ajax.systems/ua](https://ajax.systems/ua) |
| Референс шрифтов | [ajax.systems/ua/branding-guidelines/fonts/](https://ajax.systems/ua/branding-guidelines/fonts/) |
| Референс контента | [ua.goodtransformer.com/products](https://ua.goodtransformer.com/products) |

## 2. Дизайн

- Современный минимализм в стиле **Ajax Systems** — много воздуха, акцент на продукте и типографике
- Строгий деловой тон без декоративных излишеств
- Цвета: **белый фон** + **изумрудный градиент** для акцентов, CTA и активных состояний
- Шрифты — по референсу Ajax (Inter или аналог) с поддержкой кириллицы
- Переключатель темы: **светлая / тёмная / системная**. Выбор сохраняется в `localStorage`
- Полная адаптивность: мобильные, планшеты, десктопы (mobile-first)
- Логотип Zelect используется в шапке и подвале. Favicon — на основе знака «Z»

## 3. Структура сайта

### Главное меню (хедер)

- **Продукти** — каталог с категориями
- **Послуги** — услуги компании
- **Підтримка** — документация, FAQ, гарантия
- **Новини** — раздел новостей
- **Контакти** — контактная информация и форма связи

В шапке также: логотип (слева, ссылка на главную), переключатель темы, CTA-кнопка «Запит КП».

### Страницы сайта

- **Главная** `/` — hero-блок, категории, преимущества, краткое о компании, форма связи
- **Каталог** `/produkty` — список категорий
- **Категория** `/produkty/[category]` — список товаров
- **Товар** `/produkty/[category]/[product]` — детальная карточка
- **Услуги** `/poslugy` — общий список, детальные страницы `/poslugy/[slug]`
- **Поддержка** `/pidtrymka` — документация, FAQ, гарантия
- **Новости** `/novyny` — список и детальные публикации `/novyny/[slug]`
- **Контакты** `/kontakty` — данные, форма, карта
- **Служебные**: `404`, политика конфиденциальности, cookie-баннер

### Подвал (футер)

- Логотип и краткое описание компании
- Дублирование основной навигации
- Контактные данные (телефон, email, адрес)
- Юридическая информация (реквизиты, политика конфиденциальности)
- Копирайт

## 4. Функциональные требования

### 4.1. Каталог

- Иерархические категории (до 2 уровней вложенности)
- Карточка товара: название, артикул, галерея изображений, описание, технические характеристики, прикреплённые документы (PDF)
- Кнопка **«Запит ціни»** на карточке товара — открывает форму с автоподстановкой названия
- Хлебные крошки (breadcrumbs)

### 4.2. CMS (Payload)

- Админка для управления категориями, товарами, услугами, новостями, страницами
- Загрузка изображений и документов с автоматической оптимизацией
- Rich-text редактор для описаний (Lexical)
- SEO-поля (`title`, `description`) для каждой сущности
- Коллекции: `Categories`, `Products`, `Services`, `News`, `Pages`, `Media`, `Users`, `Settings` (global), `Submissions` (для заявок с форм)

### 4.3. Формы и обратная связь

- Форма **«Запит комерційної пропозиції»**: ФИО, компания, телефон, email, сообщение
- Контактная форма на странице «Контакти»
- Заявки сохраняются в CMS (коллекция `Submissions`) и отправляются на email администратора
- Защита от спама: **honeypot + rate limiting** (без reCAPTCHA — экономия)
- Валидация полей на клиенте и на сервере

### 4.4. Темы и язык

- Переключатель **светлая / тёмная / системная** тема
- В первой версии активен только украинский язык

**Архитектурная готовность к мультиязычности:** поля Payload настроены с `localized: true`. Добавление других языков (`ru`, `en`, `pl`) в будущем не требует переделки кода — только добавление переводов в админке. URL-структура предусматривает префикс языка (в первой версии можно опустить префикс для `uk`).

### 4.5. Готовность к корзине (на будущее)

В коллекции `Products` **заранее предусмотрены поля**:
- `price` (число)
- `currency` (selector: UAH / USD / EUR)
- `stock` (число / selector: в наличии / под заказ)
- `minOrderQty` (число)

В первой версии эти поля **скрыты в UI админки** (или admin-only), на фронтенде не отображаются. Структура данных и страница товара готовы к подключению модуля корзины в будущем без переработки основы.

## 5. SEO и производительность

- Динамические meta-теги (`title`, `description`) — формируются из CMS
- Open Graph для корректного отображения в соцсетях
- Семантическая HTML-разметка, `alt`-атрибуты, корректные заголовки
- ЧПУ (человекопонятные URL) для всех страниц
- Автогенерация `sitemap.xml` и `robots.txt`
- Оптимизация изображений через Next.js Image (WebP, lazy loading)
- HTTPS (Let's Encrypt), базовые security headers
- Google Analytics 4 — установка и базовая настройка
- Cookie-баннер с согласием

## 6. Контент и наполнение

Базовое наполнение каталога берётся с референса [ua.goodtransformer.com/products](https://ua.goodtransformer.com/products) — категории, список товаров, структура характеристик.

### Объём первичного наполнения

| Сущность | Количество |
|---|---|
| Категории | 10–15 |
| Товары | 50–80 |
| Услуги | 5–8 |
| Новости | 3–5 стартовых публикаций |
| Статичные страницы | Главная, О компании, Контакты, Политика конфиденциальности |

Тексты о компании, описания услуг и изображения предоставляются заказчиком. Технические характеристики товаров берутся с референса и адаптируются.

## 7. Инфраструктура и деплой

- **Хостинг:** VPS (Ubuntu)
- **Process manager:** PM2
- **Reverse proxy:** Nginx + HTTPS (Let's Encrypt)
- **База данных:** PostgreSQL
- **Хранилище медиа:** локальное на VPS (с возможностью миграции на S3-совместимое в будущем)
- **Git:** GitHub с публичным/приватным репозиторием

| 1 | Подготовка: согласование структуры, настройка коллекций Payload, базовый дизайн-каркас | 3–4 дня | Утверждённая структура |

## 9. Что не входит в первую версию

Для прозрачности договорённостей — функционал, **не входящий** в эту итерацию:

- Корзина и оформление заказов (в v1 — только запрос КП через форму)
- Онлайн-оплата
- Личный кабинет / B2B-портал
- Дополнительные языки (техническая готовность есть, активация и перевод — отдельный этап)
- Сложный поиск и фасетная фильтрация
- Чат-бот и онлайн-консультант
- Профессиональная фотосъёмка продукции
- Копирайтинг продающих текстов
- SEO-продвижение и контекстная реклама
